## Design Decisions
- **Prisma v6, not v7:** The guideline requires `url = env("DATABASE_URL")` inside the `datasource` block. Prisma 7 removed schema-level `url` (moved to `prisma.config.ts`). Pinned to `prisma@^6.16.2` so the schema matches the required format exactly. Wave 2 should stay on v6 unless a config-file migration is coordinated.
- **Named relations** (`WorkspaceOwner`, `TaskCreator`, `TaskAssignee`, `AttachmentUploader`) resolve the multiple `User` → same-model ambiguities Prisma requires.
- **Cascade vs SetNull:** Deleting a `Workspace` cascades to tasks/channels/members; deleting a `User` cascades owned data. `Task.assigneeId` uses `onDelete: SetNull` so unassigning does not delete the task.
- **snake_case table names** via `@@map` to keep the SQL layer conventional while retaining camelCase in TS.
- **Polymorphic-ish Attachment:** optional `taskId?`/`messageId?` lets one attachment model serve both tasks and messages without a join table.

5. Enums are exported from `@prisma/client` — use them for request validation (e.g., `TaskStatus`, `WorkspaceRole`).
